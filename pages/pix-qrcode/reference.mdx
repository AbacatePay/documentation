## Criar uma cobrança com checkout transparente

Use `/transparents/create` para criar um QRCode PIX. A API retorna um código Base64 para gerar o QRCode e um código copia-e-cola PIX que pode ser enviado ao seu cliente.

<Card title="Campos obrigatórios" icon="info" horizontal>
  Apenas `amount` é obrigatório. O valor deve ser informado em centavos.
</Card>

**Exemplo:**
```json
POST /transparents/create
{
  "amount": 10000,                              // obrigatório - valor em centavos
}
```
<Tip title="Apenas o amount é obrigatório" icon="info" horizontal>
   Apenas `amount` é obrigatório. O valor deve ser informado em centavos.
</Tip>

**Resposta:**
```json
{
  "data": {
    "id": "pix_abc123xyz",
    "amount": 10000,
    "status": "PENDING",
    "devMode": false,
    "brCode": "00020160014BR.GOV.BCB.PIX070503***6304ABCD",
    "brCodeBase64": "data:image/png;base64,iVBORw0KG...",
    "platformFee": 100,
    "createdAt": "2024-11-04T18:38:28.573Z",
    "updatedAt": "2024-11-04T18:38:28.573Z",
    "expiresAt": "2024-11-04T19:38:28.573Z"
  },
  "error": null
}
```

<Tip title="Use o QRCode retornado" icon="book" horizontal>
Use o `brCodeBase64` para gerar a imagem do QRCode ou o `brCode` (código copia-e-cola) para enviar ao seu cliente via WhatsApp, Telegram ou integrar no seu checkout.
</Tip>

## 💡 Dicas e Ideias de Uso

Esta rota de checkout transparente oferece diversas possibilidades de integração:

- **Venda via WhatsApp e Telegram**: Você pode gerar o QRCode e o código copia-e-cola e enviar diretamente ao seu cliente através de mensagens no WhatsApp ou Telegram, facilitando o processo de pagamento.

- **Checkout próprio na sua aplicação**: Crie uma experiência de checkout totalmente personalizada e integrada ao design da sua aplicação. Isso permite maior controle sobre a interface, melhor experiência do usuário e a possibilidade de manter seu cliente dentro do seu ambiente, aumentando a conversão e fortalecendo sua marca.

<Info>
Por enquanto, esta rota aceita apenas o método de pagamento **PIX**.
</Info>

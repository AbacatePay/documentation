Ir para produção envolve dois passos independentes: **verificar sua conta** (processo burocrático, feito uma vez só) e **trocar a chave de API** no seu sistema (mudança de código). Ambos precisam estar prontos antes de processar pagamentos reais.

<Info>
  Antes de iniciar, confirme que você passou pelo [checklist do Dev mode](/pages/devmode#checklist-antes-de-ir-para-producao). Ir para produção com integração incompleta pode causar falhas silenciosas.
</Info>

## O que muda na prática

| | Dev mode | Produção |
|---|---|---|
| Chave de API | Começa com `dev_` | Começa com `prod_` |
| Pagamentos | Simulados | Reais |
| Webhooks | Eventos fictícios | Eventos reais |
| Saque | Não disponível | Disponível |
| Aprovação necessária | Não | Sim |

A troca de ambiente é só isso: substituir a chave de API. Não há URL diferente.

Al migrar do ambiente de desenvolvimento para produção, você precisará completar o processo de verificação da sua conta. Este processo é necessário para garantir a segurança e conformidade das operações.

<Card title="Validação do Projeto" horizontal>

  Antes de iniciar o processo, verifique se a sua operação está na nossa [lista de operações suportadas](/operacoes-suportadas) para garantir que poderemos dar suporte ao seu negócio.

## Após a aprovação

Quando sua conta for aprovada, você receberá um e-mail com as instruções. O que fazer:

1. Gere uma chave de **Produção** no dashboard (Integração → API Keys → Criar Chave)
2. Substitua a chave de Dev mode no seu sistema pela chave de Produção
3. Crie novamente os webhooks no ambiente de Produção (webhooks de Dev não recebem eventos reais)
4. Faça uma transação de teste com valor real para confirmar que tudo está funcionando

<Warning>
  Não reutilize webhooks do Dev mode em produção. Eles precisam ser criados separadamente com a chave de Produção.
</Warning>

Flags disponíveis em todos os comandos:

| Flag | Alias | Tipo | Padrão | Descrição |
|------|-------|------|--------|-----------|
| `--verbose` | `-v` | bool | `false` | Habilita logging detalhado (nível debug) |
| `--local` | `-l` | bool | `false` | Usa servidor de teste |
| `--output` | `-o` | string | `text` | Formato de saída: `text`, `json`, `table` |

```bash
# Exemplo combinando flags
abacatepay -v -l -o json payments create pix
```

---

A CLI suporta três formatos de saída, configuráveis via `--output` ou `-o`.

### text (padrão)

Formato legível para uso interativo no terminal.

```bash
abacatepay whoami
```

```text
✓ User Information
  Profile: default
  User: João Silva
  Email: joao@empresa.com
  Status: Authenticated
```

### json

Formato estruturado para scripts, pipelines e CI/CD.

```bash
abacatepay -o json whoami
```

```json
{
  "profile": "default",
  "user": "João Silva",
  "email": "joao@empresa.com",
  "status": "authenticated"
}
```

Combine com `jq` para filtrar dados:

```bash
abacatepay -o json logs list | jq '.logs[] | select(.msg=="webhook_forwarded")'
```

### table

Formato tabular para visualização de listas.

```bash
abacatepay -o table profile list
```

```text
┌───────────┬────────────┬────────┐
│ Name      │ API Key    │ Active │
├───────────┼────────────┼────────┤
│ default   │ abc_prod...│ 🥑     │
│ staging   │ abc_dev... │        │
└───────────┴────────────┴────────┘
```

## CLI Reference

### Autenticação

Comandos para gerenciar a autenticação e sessão na AbacatePay CLI.

---

#### `abacatepay login`

Realiza a autenticação na AbacatePay. Abre o navegador para login via OAuth ou permite autenticação direta com API key.

```bash
abacatepay login [flags]
```

| Flag | Alias | Descrição | Padrão |
|------|-------|-----------|--------|
| `--name` | - | Nome do perfil a ser criado | `""` |
| `--key` | - | Sua API key da AbacatePay | `""` |
| `-l` | `--local` | Usa servidor de teste | `false` |

<Tip>
Use `--name` para criar múltiplos perfis e alternar entre diferentes contas ou ambientes.
</Tip>

**Exemplos:**

```bash
# Login interativo via navegador
abacatepay login

# Login com API key específica
abacatepay login --key "abc_prod_xxxxx"

# Login criando um perfil nomeado
abacatepay login --name producao --key "abc_prod_xxxxx"
```

### Pagamentos

Os comandos de pagamentos permitem criar, verificar e simular cobranças diretamente pela linha de comando.

#### `abacatepay payments create`

Cria uma nova cobrança de pagamento. Suporta PIX QR Code e Checkout.

Por padrão, cria um pagamento com dados fictícios (mock). Use o modo interativo para especificar os detalhes manualmente.

```bash
abacatepay payments create [pix|checkout] [flags]
```

| Flag | Alias | Descrição | Padrão |
|------|-------|-----------|--------|
| `--interactive` | `-i` | Ativa o modo interativo para inserir dados | `false` |

<Tip>
Se você não especificar o método (`pix` ou `checkout`), a CLI exibirá um menu interativo para seleção.
</Tip>

**Exemplos:**

```bash
# Criar PIX com dados mock
abacatepay payments create pix

# Criar checkout no modo interativo
abacatepay payments create checkout -i

# Selecionar método interativamente
abacatepay payments create
```

### Eventos & Webhooks

Comandos para escutar, simular e depurar webhooks durante o desenvolvimento.

---

#### `abacatepay listen`

Escuta webhooks em tempo real e os encaminha para sua aplicação local.

```bash
abacatepay listen [flags]
```

| Flag | Alias | Descrição | Padrão |
|------|-------|-----------|--------|
| `--forward-to` | - | URL para onde os eventos serão encaminhados | `http://localhost:3000/api/webhooks` |
| `--mock` | - | Simula webhooks sem conectar à API | `false` |

<Info>
O comando `listen` mantém uma conexão WebSocket ativa com a AbacatePay para receber eventos em tempo real. Todos os eventos recebidos são salvos no log local.

Se você não especificar `--forward-to`, a CLI perguntará via prompt qual URL usar, com `http://localhost:3000/webhooks/abacatepay` como padrão.
</Info>

**Exemplos:**

```bash
# Escutar e encaminhar para o endpoint padrão
abacatepay listen

# Encaminhar para URL personalizada
abacatepay listen --forward-to http://localhost:8080/webhook

# Modo simulação (sem conexão com a API)
abacatepay listen --mock
```

<Tip>
Use `Ctrl+C` para interromper. Todos os eventos ficam salvos e podem ser consultados com `abacatepay logs list`.
</Tip>

---

### Utilitários

Comandos auxiliares para verificação de webhooks, atualização e documentação.

---

#### `abacatepay verify`

Verifica assinaturas de webhooks localmente, útil para debug e validação offline.

```bash
abacatepay verify --secret <secret> --payload <json> --signature <header>
```

| Flag | Tipo | Obrigatória | Descrição |
|------|------|-------------|-----------|
| `--secret` | string | Sim | Webhook signing secret (inicia com `whsec_`) |
| `--payload` | string | Sim | Corpo JSON raw do payload |
| `--signature` | string | Sim | Valor do header `X-Abacate-Signature` |

<Info>
O formato do header de assinatura segue o padrão: `t=TIMESTAMP,v1=SIGNATURE`
</Info>

```bash
abacatepay verify \
  --secret "whsec_abc123xyz789" \
  --payload '{"id":"evt_123","event":"billing.paid"}' \
  --signature "t=1705849200,v1=a1b2c3d4e5f6..."
```

<CodeGroup>
```text Sucesso
╭───────────────────────────╮
│                           │
│  🥑 Signature Verified    │
│                           │
│  Timestamp: 1705849200    │
```

## Casos de Uso

### Teste rápido de integração local

```bash
# 1. Autentique em modo teste
abacatepay -l login

# 2. Inicie o listener
abacatepay -l listen --forward-to http://localhost:3000/webhooks

# 3. Em outro terminal, crie uma cobrança e simule o pagamento
abacatepay -l payments create pix
abacatepay -l payments simulate pix_xxx
```

### Debug de webhooks

```bash
# Capture com verbose para ver detalhes
abacatepay -v listen --forward-to http://localhost:3000/webhooks

# Verifique a assinatura manualmente se falhar
abacatepay verify \
  --secret "whsec_seu_secret" \
  --payload '{"id":"evt_123",...}' \
  --signature "t=123456,v1=abc..."
```

### Automação em CI/CD

```bash
# Output JSON para scripts
PAYMENT_ID=$(abacatepay -o json payments create pix | jq -r '.data.id')

# Simular e verificar
abacatepay payments simulate $PAYMENT_ID
STATUS=$(abacatepay -o json payments check $PAYMENT_ID | jq -r '.data.status')

if [ "$STATUS" = "PAID" ]; then
  echo "Pagamento confirmado!"
fi
```

### Múltiplas contas/ambientes

```bash
# Criar perfis diferentes
abacatepay login --name producao --key "abc_prod_xxx"
abacatepay login --name sandbox --key "abc_dev_xxx"

# Alternar entre perfis
abacatepay switch producao
abacatepay whoami

abacatepay switch sandbox
abacatepay -l listen
```

## Ambientes

A CLI suporta dois ambientes: **produção** (padrão) e **teste**.

### Produção (padrão)

```bash
abacatepay login
```

Conecta automaticamente com `wss://ws.abacatepay.com/ws`.

### Servidor de Teste

```bash
abacatepay login -l
```

Conecta com o servidor de teste em `ws://191.252.202.128:8080/ws`.

<Warning>
O ambiente de teste usa dados fictícios. Nunca use credenciais reais no modo `-l`.
</Warning>

## Armazenamento Seguro

O token da sua conta é armazenado no keyring nativo do sistema:

- **macOS**: Keychain
- **Windows**: Credential Manager
- **Linux**: gnome-keyring ou kwallet

<Accordion title="Problema ao salvar token no Linux?">
Instale o keyring no seu sistema:

```bash
# Debian/Ubuntu
sudo apt install gnome-keyring

# Fedora
sudo dnf install gnome-keyring
```

Então tente novamente:

```bash
abacatepay login
```
</Accordion>

## Logs & Debugging

Os logs são salvos em `~/.abacatepay/logs/`:

- **abacatepay.log** - Log geral (JSON)
- **transactions.log** - Webhooks recebidos e encaminhados

Rotação automática: 10MB por arquivo, 5 backups, 30 dias de retenção.

```bash
# Ver erros
cat ~/.abacatepay/logs/abacatepay.log | jq 'select(.level=="ERROR")'

# Ver webhooks recebidos
cat ~/.abacatepay/logs/transactions.log | jq 'select(.msg=="webhook_received")'

# Tempo médio de encaminhamento
cat ~/.abacatepay/logs/transactions.log | jq 'select(.msg=="webhook_forwarded") | .duration_ms' | jq -s 'add/length'
```

## Perguntas Frequentes

<Accordion title="A CLI é indicada para produção?">
A CLI é voltada principalmente para desenvolvimento, testes e automação. Para aplicações em produção, recomendamos os SDKs ou integração direta com a API REST.
</Accordion>

<Accordion title="Funciona em CI/CD?">
Sim. A CLI é totalmente não-interativa quando usada com flags. Use `-o json` para output parseável e variáveis de ambiente para credenciais.
</Accordion>

<Accordion title="A CLI substitui os SDKs?">
Não. Ela complementa os SDKs, oferecendo uma interface direta via terminal para desenvolvimento e debugging.
</Accordion>

<Accordion title="Como verificar se estou autenticado?">
Use `abacatepay status` ou `abacatepay whoami` para verificar o status da autenticação e qual perfil está ativo.
</Accordion>

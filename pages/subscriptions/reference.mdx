Uma **assinatura** na AbacatePay é um **Checkout** para cobrança recorrente: a mesma página de pagamento do Checkout comum, com os mesmos parâmetros (`returnUrl`, `completionUrl`, `customerId`, `externalId`, `metadata`, `coupons`, `method`). A diferença é que o Checkout de assinatura aceita **apenas um produto**, e esse produto deve ter sido **criado com ciclo (frequency)** já definido na loja.

## Parâmetros (igual ao Checkout)

O `POST /subscriptions/create` aceita os mesmos parâmetros do [Checkout](/pages/payment/reference):

| Parâmetro       | Obrigatório | Descrição |
|-----------------|-------------|-----------|
| `items`         | Sim         | Lista com **exatamente um** item (`id` e `quantity`); o produto deve ter sido criado com ciclo de assinatura |
| `customerId`    | Não         | ID do cliente já cadastrado |
| `externalId`    | Não         | ID da assinatura no seu sistema |
| `returnUrl`     | Não         | URL de retorno ao clicar em "Voltar" |
| `completionUrl` | Não         | URL após pagamento concluído |
| `metadata`      | Não         | Metadados livres |
| `coupons`       | Não         | Lista de cupons |
| `method`        | Não         | `PIX` ou `CARD` |

Exemplo de body (apenas `id` e `quantity` no item; o ciclo já está no produto):

```json
{
  "items": [
    {
      "id": "prod-1234",
      "quantity": 1
    }
  ],
  "customerId": "cust_abc123xyz",
  "externalId": "subs-123",
  "returnUrl": "https://seusite.com/voltar",
  "completionUrl": "https://seusite.com/sucesso"
}
```

## Resposta (igual ao Checkout)

A resposta do create e os itens do list seguem o mesmo payload do Checkout: `id`, `url`, `amount`, `items`, `status`, `returnUrl`, `completionUrl`, `customerId`, `metadata`, `createdAt`, `updatedAt`, etc.

**Exemplo de resposta do create:**

```json
{
  "data": {
    "id": "bill_abc123xyz",
    "externalId": "subs-123",
    "url": "https://app.abacatepay.com/pay/bill_abc123xyz",
    "amount": 10000,
    "paidAmount": null,
    "items": [
      {
        "id": "prod_456",

      }
    ],
    "status": "PENDING",
```

## Ciclo de cobrança (frequency)

O ciclo da assinatura é definido **no produto** ao criá-lo na loja. Valores possíveis: **MONTHLY**, **YEARLY**, **WEEKLY**, **DAILY**.

## Status do Checkout

Os mesmos status do Checkout se aplicam: **PENDING**, **EXPIRED**, **CANCELLED**, **PAID**, **REFUNDED**.

## Segurança

- Todas as requisições são autenticadas via Bearer Token
- Abusos desta API podem resultar em suspensão da conta conforme descrito em nossos [termos de uso](https://abacatepay.com/termos)

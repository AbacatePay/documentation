Use `/customers/create` para criar um cliente antes da cobrança. Útil quando você já conhece os dados do cliente ou quer reutilizar o `customerId` em múltiplas cobranças.

<Card title="Campos obrigatórios" icon="info" horizontal>
  Apenas `email` é obrigatório. Recomendamos também enviar `name`, `cellphone` e `taxId` quando disponíveis.
</Card>

**Exemplo:**
```json
POST /customers/create
{
  "data": {
    "email": "customer@example.com",        // obrigatório
    "taxId": "12345678900",                 // opcional
    "name": "João Silva",                   // opcional
    "cellphone": "+5511999999999",          // opcional
    "zipCode": "01310-100"                  // opcional
  },
  "metadata": {                             // Aqui você pode adicionar dados adicionais da sua aplicação
    "customField": "value"
  }
}
```

**Resposta:**
```json
{
  "data": {
    "id": "cust_abc123xyz",
    "devMode": false,
    "email": "customer@example.com",
    "taxId": "12345678900",
    "country": "BR",
    "name": "João Silva",
    "cellphone": "+5511999999999",
    "zipCode": "01310-100",
    "metadata": {
      "customField": "value"
    }
  },
  "error": null
}
```

O `customerId` retornado pode ser usado ao criar cobranças, pré-preenchendo os dados do checkout.

**Alternativa**: O cliente também pode ser criado automaticamente ao criar uma cobrança via `/checkouts/create` ou quando o cliente preenche os dados no checkout.

<Tip title="Clientes são únicos por CPF/CNPJ" icon="book" horizontal>
 Não é possível criar um cliente com CPF/CNPJ inválido e caso já exista um cliente com esse CPF/CNPJ, a API não criará o cliente novo mas retornará o cliente já existente
</Tip>
